# 项目同步说明

这个仓库建议使用双远程：

- `upstream`：原作者仓库，用来拉取作者更新
- `origin`：你自己的 GitHub 仓库，用来保存你的版本、部署和绑定域名

当前已把原作者仓库设置为：

```bash
upstream https://github.com/hero8152/Infinite-Canvas.git
```

## 第一次绑定你自己的仓库

先在 GitHub 创建一个空仓库，然后执行：

```bash
git remote add origin https://github.com/你的用户名/你的仓库名.git
git push -u origin main
```

不要把 `data/`、`assets/`、日志、API Key、本地 Python 依赖提交上去；这些已经写进 `.gitignore`。

## 以后同步原作者更新

不需要重新下载压缩包，也不需要手动上传覆盖。直接执行：

```bash
git fetch upstream
git merge upstream/main
```

如果没有冲突，再推到你自己的仓库：

```bash
git push origin main
```

## 如果有冲突

说明你改过的文件和作者更新的文件撞到一起了。处理方式：

```bash
git status
```

打开冲突文件，保留需要的内容，删掉冲突标记：

```text
<<<<<<< HEAD
你的版本
=======
作者版本
>>>>>>> upstream/main
```

处理完后：

```bash
git add .
git commit
git push origin main
```

## 部署和域名

这个项目是 FastAPI 后端，不适合直接用 GitHub Pages。

推荐部署到 VPS、宝塔、Railway、Render、Zeabur 等能运行 Python 服务的平台，然后让域名解析到服务器，再用 Nginx 或 Caddy 反代到项目的 `3000` 端口。
